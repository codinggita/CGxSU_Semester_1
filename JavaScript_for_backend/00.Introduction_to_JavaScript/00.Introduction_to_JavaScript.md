## What is JavaScript?

### Formal Definition

**JavaScript** is a high-level, dynamically-typed, interpreted programming language that conforms to the ECMAScript specification. It is characterized by:

- **High-level:** Provides abstractions from hardware details (automatic memory management, garbage collection)
- **Dynamically-typed:** Type checking occurs at runtime; variables can hold values of any type
- **Interpreted:** Code is executed line-by-line by a JavaScript engine (though modern engines use JIT compilation)
- **Multi-paradigm:** Supports imperative, functional, and object-oriented programming styles
- **Single-threaded with asynchronous capabilities:** Executes one operation at a time but supports non-blocking I/O via the event loop

### Historical Context

**Created:** 1995 by Brendan Eich at Netscape
**Original Purpose:** Add interactivity to web pages
**Current Scope:** Universal programming language for web, server, mobile, desktop, IoT

### Why JavaScript Matters

JavaScript is the **only programming language** natively supported by all web browsers, making it essential for:

1. **Frontend development:** User interfaces, single-page applications (SPAs)
2. **Backend development:** Server-side logic with Node.js
3. **Full-stack development:** Same language for client and server
4. **Cross-platform applications:** Desktop (Electron), mobile (React Native), IoT

**Market Position (2025):**
- Most popular programming language (Stack Overflow Survey)
- Largest package ecosystem (npm with 2M+ packages)
- Highest job demand for web development roles

## JavaScript's Execution Environments

JavaScript runs in two primary environments, each with distinct capabilities:

### 1. Browser Environment (Client-Side)

**JavaScript Engine:** Each browser includes a JavaScript engine

| Browser | Engine | Company |
|---------|--------|---------|
| Chrome | V8 | Google |
| Firefox | SpiderMonkey | Mozilla |
| Safari | JavaScriptCore (Nitro) | Apple |
| Edge | V8 (Chromium-based) | Microsoft |

**Capabilities:**
- DOM manipulation (modify HTML/CSS)
- Event handling (clicks, keyboard, mouse)
- Browser APIs (Fetch, LocalStorage, Geolocation, WebRTC)
- Rendering UI components

**Restrictions:**
- No direct file system access (security sandbox)
- No direct OS-level operations
- Same-origin policy (security constraint)

### 2. Node.js Environment (Server-Side)

**JavaScript Engine:** V8 (same as Chrome)
**Runtime:** Node.js provides additional APIs beyond browser capabilities

**Capabilities:**
- File system operations (read/write files)
- Network operations (HTTP servers, TCP/UDP sockets)
- OS-level operations (process management, system info)
- Database connections
- Command-line utilities

**Restrictions:**
- No DOM or browser-specific APIs
- No window, document, or navigator objects

## Core Language Characteristics

### 1. Dynamic Typing

Variables do not have fixed types; type is determined at runtime based on the assigned value.

```javascript
let value = 42;              // Type: number
console.log(typeof value);   // Output: "number"

value = "Hello";             // Type: string (reassignment allowed)
console.log(typeof value);   // Output: "string"

value = true;                // Type: boolean
console.log(typeof value);   // Output: "boolean"

value = null;                // Type: object (historical quirk)
console.log(typeof value);   // Output: "object"
```

**Trade-offs:**
- âœ… **Advantage:** Faster development, less boilerplate code
- âŒ **Disadvantage:** Type-related errors occur at runtime, not compile-time
- ðŸ› ï¸ **Mitigation:** Use TypeScript, JSDoc annotations, or runtime validation

### 2. Event-Driven Architecture

JavaScript responds to events (user actions, timers, network responses) using an **event loop** and **callback functions**.

```javascript
// Browser example: respond to button click
document.getElementById('myButton').addEventListener('click', () => {
  console.log('Button clicked!');
});

// Node.js example: respond to HTTP request
const http = require('http');
http.createServer((request, response) => {
  response.end('Request received!');
}).listen(3000);
```

**Key Concept:** Code doesn't run top-to-bottom linearly; instead, it registers event listeners and waits for events to occur.

### 3. Single-Threaded Execution

JavaScript runs on a **single thread** (one operation at a time) but uses an **asynchronous, non-blocking** model to handle concurrent operations.

**Event Loop Mechanism:**
1. Execute synchronous code
2. Check callback queue for pending async operations
3. Execute callbacks one at a time
4. Repeat

**Example:**
```javascript
console.log('1: Start');

setTimeout(() => {
  console.log('2: Async callback');
}, 0);

console.log('3: End');

// Output order:
// 1: Start
// 3: End
// 2: Async callback
```

Even though `setTimeout` has 0ms delay, it runs **after** synchronous code completes.

### 4. First-Class Functions

Functions are **first-class citizens:** they can be assigned to variables, passed as arguments, and returned from other functions.

```javascript
// Function assigned to variable
const greet = function(name) {
  return `Hello, ${name}!`;
};

// Function passed as argument (callback)
const execute = (fn, value) => fn(value);
console.log(execute(greet, 'Alice')); // Output: Hello, Alice!

// Function returned from function (higher-order function)
const createMultiplier = factor => num => num * factor;
const double = createMultiplier(2);
console.log(double(5)); // Output: 10
```

### 5. Prototype-Based Inheritance

JavaScript uses **prototypes** for inheritance, not classical classes (though ES6 added `class` syntax as syntactic sugar).

```javascript
// Every object has a prototype
const obj = {};
console.log(obj.__proto__ === Object.prototype); // true

// Inheritance through prototype chain
const parent = { species: 'Mammal' };
const child = Object.create(parent);
child.name = 'Dog';

console.log(child.name);    // Own property: "Dog"
console.log(child.species); // Inherited: "Mammal"
```

## JavaScript Versions & ECMAScript

### ECMAScript (ES) Standard

**ECMAScript** is the official specification for JavaScript, maintained by ECMA International (Technical Committee 39).

**Release Cycle:** Annual releases since ES2015 (ES6)

### Major Versions

| Version | Year | Key Features |
|---------|------|--------------|
| **ES3** | 1999 | Regular expressions, try/catch |
| **ES5** | 2009 | Strict mode, JSON, Array methods (map, filter, reduce) |
| **ES6 (ES2015)** | 2015 | let/const, arrow functions, classes, modules, promises, template literals |
| **ES2016** | 2016 | `Array.prototype.includes`, exponentiation operator (`**`) |
| **ES2017** | 2017 | async/await, Object.entries/values, string padding |
| **ES2018** | 2018 | Rest/spread for objects, async iteration |
| **ES2019** | 2019 | Array.flat/flatMap, Object.fromEntries |
| **ES2020** | 2020 | Optional chaining (`?.`), nullish coalescing (`??`), BigInt |
| **ES2021** | 2021 | Logical assignment operators, WeakRef |
| **ES2022** | 2022 | Top-level await, class fields, `at()` method |
| **ES2023** | 2023 | Array methods (toSorted, toReversed, with), hashbang |

### Browser Compatibility

Modern browsers support ES2015+ features. For older browsers, use **transpilers** (Babel) to convert modern JavaScript to ES5.

**Check compatibility:** [caniuse.com](https://caniuse.com) or [MDN Browser Compatibility](https://developer.mozilla.org/en-US/docs/Web/JavaScript)

### This Curriculum's Approach

We use **ES2020+ features** as the baseline, with notes about compatibility when relevant.

## Running JavaScript Code

### Method 1: Browser Console (Interactive REPL)

**Steps:**
1. Open browser (Chrome/Firefox)
2. Press `F12` or `Ctrl+Shift+I` (Windows/Linux) / `Cmd+Option+I` (Mac)
3. Navigate to **Console** tab
4. Type JavaScript code and press Enter

**Example:**
```javascript
console.log('Hello from browser console!');
// Output: Hello from browser console!

const sum = (a, b) => a + b;
sum(5, 3);
// Output: 8
```

**Use Case:** Quick testing, debugging, exploring APIs

### Method 2: HTML `<script>` Tag (Inline)

**Example:**
```html
<!DOCTYPE html>

  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

      alert('Button was clicked!');
    });
  </script>

</html>
```

**Use Case:** Small scripts, prototyping, learning exercises

### Method 3: External JavaScript File (Recommended)

**HTML file (index.html):**
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>External JavaScript</title>
</head>

  <h1>External JavaScript Example</h1>
  <div id="output"></div>

  <!-- Load script at end of body (DOM ready) -->
  <script src="app.js"></script>

```

**JavaScript file (app.js):**
```javascript
// app.js

output.textContent = 'JavaScript loaded successfully!';

```

**Benefits:**
- âœ… Separation of concerns (HTML structure, JS behavior)
- âœ… Code reusability across multiple HTML pages
- âœ… Browser caching (faster subsequent loads)
- âœ… Easier maintenance and version control

**Best Practice:** Place `<script>` tags at the **end of `<body>`** (ensures DOM is loaded) or use `defer` attribute:

```html

```

**JavaScript (counter.js):**
```javascript
/**
 * Event-Driven Counter Application
 *
 * Demonstrates:
 * - Event listeners and handlers
 * - DOM manipulation
 * - State management
 * - Function composition
 */

// --- State management ---
let count = 0;

// --- DOM element references ---
const counterDisplay = document.getElementById('counter');
const incrementBtn = document.getElementById('increment');
const decrementBtn = document.getElementById('decrement');
const resetBtn = document.getElementById('reset');

// --- Update UI function ---
const updateDisplay = () => {
  counterDisplay.textContent = count;
  counterDisplay.style.color = count > 0 ? 'green' : count < 0 ? 'red' : '#333';
};

// --- Event handlers ---
const increment = () => {
  count++;
  updateDisplay();
  console.log(`Incremented: ${count}`);
};

const decrement = () => {
  count--;
  updateDisplay();
  console.log(`Decremented: ${count}`);
};

const reset = () => {
  count = 0;
  updateDisplay();
  console.log('Reset to 0');
};

// --- Attach event listeners ---
incrementBtn.addEventListener('click', increment);
decrementBtn.addEventListener('click', decrement);
resetBtn.addEventListener('click', reset);

// --- Initialize display ---
updateDisplay();

// --- Export for testing (Node.js environment simulation) ---
if (typeof module !== 'undefined' && module.exports) {
  module.exports = { increment, decrement, reset, getCount: () => count };
}
```

## Common Pitfalls & Debugging

### Pitfall 1: `typeof null` Returns `"object"`

**Problem:**
```javascript
typeof null; // "object" (not "null"!)
```

**Explanation:** Historical JavaScript bug that cannot be fixed (breaking change)

**Solution:**
```javascript
const isNull = value => value === null;
const getType = value => value === null ? 'null' : typeof value;
```

---

### Pitfall 2: Automatic Semicolon Insertion (ASI)

**Problem:**
```javascript
function getValue() {
  return
    { value: 42 }; // Returns undefined! (ASI inserts semicolon after return)
}
```

**Solution:**
```javascript
function getValue() {
  return { // Opening brace on same line
    value: 42,
  };
}
```

---

### Pitfall 3: `var` Hoisting and Scope Issues

**Problem:**
```javascript
console.log(x); // undefined (not ReferenceError)
var x = 5;

// Equivalent to:
var x;
console.log(x);
x = 5;
```

**Solution:** Use `const` and `let` (block-scoped, not hoisted)
```javascript
console.log(x); // ReferenceError: Cannot access 'x' before initialization
const x = 5;
```

---

### Pitfall 4: Equality Operators (`==` vs `===`)

**Problem:**
```javascript
0 == false;     // true (type coercion)
'' == false;    // true
null == undefined; // true
'5' == 5;       // true
```

**Solution:** Always use strict equality (`===`) and inequality (`!==`)
```javascript
0 === false;    // false
'' === false;   // false
'5' === 5;      // false
```

```bash

# Bundle for production
```

```javascript
import * as Sentry from '@sentry/browser';

Sentry.init({
  dsn: 'your-dsn-url',
  environment: 'production',
});

try {
  riskyOperation();

  Sentry.captureException(error);
}
```

### 4. Performance Monitoring

```javascript
// Measure execution time
console.time('Operation');
```
