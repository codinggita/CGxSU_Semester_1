## Introduction

Template literals (also called template strings) were introduced in ES6 to solve several pain points with traditional string handling in JavaScript. They provide a cleaner, more powerful way to work with strings.

### Problems with Traditional Strings

```javascript
// Problem 1: Concatenation is messy and hard to read
var greeting = "Hello, " + name + "! Welcome to " + course + ".";

// Problem 2: Multiline strings require escape characters
var html = "<div>\n" +
           "  <h1>" + title + "</h1>\n" +
           "  <p>" + content + "</p>\n" +
           "</div>";

// Problem 3: Complex expressions require breaking up strings
var message = "The total is: $" + (price * quantity).toFixed(2);
```

### Template Literals Solution

```javascript
// Solution 1: Clean interpolation
const greeting = `Hello, ${name}! Welcome to ${course}.`;

// Solution 2: Natural multiline strings
const html = `
  <div>
    <h1>${title}</h1>
    <p>${content}</p>
  </div>
`;

// Solution 3: Inline expressions
const message = `The total is: $${(price * quantity).toFixed(2)}`;
```

## Basic Syntax

Template literals use **backticks** (`` ` ``) instead of quotes.

```javascript
// Traditional string
const traditional = "Hello, World!";

// Template literal
const template = `Hello, World!`;
```

**Key Point**: The backtick key is usually located below the `Esc` key on your keyboard.

### When to Use Template Literals

- When you need to include variables in strings
- When creating multiline text
- When building HTML/XML structures
- When formatting dynamic messages
- When you need to evaluate expressions within strings

## String Interpolation

String interpolation allows you to insert variables and expressions into strings using `${}`.

### Basic Variable Interpolation

```javascript
const name = "Arjun";
const age = 19;
const city = "Ahmedabad";

// Old way
const intro1 = "My name is " + name + ", I am " + age + " years old, and I live in " + city + ".";

// Template literal way
const intro2 = `My name is ${name}, I am ${age} years old, and I live in ${city}.`;

console.log(intro2);
// Output: My name is Arjun, I am 19 years old, and I live in Ahmedabad.
```

### Object Property Interpolation

```javascript
const student = {
  name: "Krishna",
  rollNo: 101,
  branch: "CSE",
  semester: 3
};

const studentInfo = `
Student Details:
Name: ${student.name}
Roll Number: ${student.rollNo}
Branch: ${student.branch}
Semester: ${student.semester}
`;

console.log(studentInfo);
```

### Array Interpolation

```javascript
const fruits = ["Apple", "Banana", "Orange"];

const message = `Available fruits: ${fruits.join(", ")}`;
console.log(message);
// Output: Available fruits: Apple, Banana, Orange

// Using array methods
const numbers = [1, 2, 3, 4, 5];
const stats = `Sum: ${numbers.reduce((a, b) => a + b)}, Count: ${numbers.length}`;
console.log(stats);
// Output: Sum: 15, Count: 5
```

## Tagged Templates

Tagged templates are an advanced feature that allows you to parse template literals with a function.

### Basic Tagged Template

```javascript
function myTag(strings, ...values) {
  console.log("Strings:", strings);
  console.log("Values:", values);

  return strings.reduce((result, str, i) => {
    return result + str + (values[i] || "");
  }, "");
}

const name = "Arjun";
const age = 19;

const result = myTag`Hello ${name}, you are ${age} years old.`;
console.log(result);

// Output:
// Strings: ["Hello ", ", you are ", " years old."]
// Values: ["Arjun", 19]
// Hello Arjun, you are 19 years old.
```

### Highlighting Values

```javascript
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    const value = values[i] !== undefined ? `<mark>${values[i]}</mark>` : "";
    return result + str + value;
  }, "");
}

const product = "Laptop";
const price = 45000;

const html = highlight`The ${product} costs â‚¹${price}`;
console.log(html);
// Output: The <mark>Laptop</mark> costs â‚¹<mark>45000</mark>
```

### SQL Query Builder (Safe Parameterization)

```javascript
function sql(strings, ...values) {
  return {
    text: strings.reduce((query, str, i) => {
      return query + str + (i < values.length ? `$${i + 1}` : "");
    }, ""),
    values: values
  };
}

const username = "arjun";
const minAge = 18;

const query = sql`
  SELECT * FROM users
  WHERE username = ${username}
  AND age >= ${minAge}
`;

console.log(query);
// Output:
// {
//   text: "SELECT * FROM users WHERE username = $1 AND age >= $2",
//   values: ["arjun", 18]
// }
```

### HTML Sanitization

```javascript
function sanitize(strings, ...values) {
  return strings.reduce((result, str, i) => {
    if (i < values.length) {
      const value = String(values[i])
        .replace(/&/g, "&amp;")
        .replace(/</g, "&lt;")
        .replace(/>/g, "&gt;")
        .replace(/"/g, "&quot;")
        .replace(/'/g, "&#x27;");
      return result + str + value;
    }
    return result + str;
  }, "");
}

const userInput = "<script>alert('XSS')</script>";
const safeHTML = sanitize`<div>${userInput}</div>`;
console.log(safeHTML);
// Output: <div>&lt;script&gt;alert(&#x27;XSS&#x27;)&lt;/script&gt;</div>
```

## Raw Strings

The `String.raw` tag function provides access to raw strings without processing escape sequences.

### Basic Raw String

```javascript
// Normal template literal processes escape sequences
const normal = `Line 1\nLine 2\tTabbed`;
console.log(normal);
// Line 1
// Line 2    Tabbed

// Raw string preserves escape sequences
const raw = String.raw`Line 1\nLine 2\tTabbed`;
console.log(raw);
// Line 1\nLine 2\tTabbed
```

### File Paths (Windows)

```javascript
// Without String.raw - backslashes are escape characters
const path1 = `C:\Users\Arjun\Documents`; // Wrong!
console.log(path1); // C:UsersArjunDocuments (broken)

// With String.raw - backslashes are preserved
const path2 = String.raw`C:\Users\Arjun\Documents`;
console.log(path2); // C:\Users\Arjun\Documents (correct)
```

### Regular Expressions

```javascript
// Easier to write regex patterns
const regex = new RegExp(String.raw`\d+\.\d+`);
console.log(regex.test("3.14")); // true
```

## Performance Considerations

### Template Literals vs Concatenation

```javascript
// Performance comparison
const iterations = 1000000;

// Test 1: Concatenation
console.time("Concatenation");
for (let i = 0; i < iterations; i++) {
  const result = "Hello, " + "World" + "!";
}
console.timeEnd("Concatenation");

// Test 2: Template Literal
console.time("Template Literal");
for (let i = 0; i < iterations; i++) {
  const result = `Hello, ${"World"}!`;
}
console.timeEnd("Template Literal");

// Test 3: Simple Template Literal (no interpolation)
console.time("Simple Template");
for (let i = 0; i < iterations; i++) {
  const result = `Hello, World!`;
}
console.timeEnd("Simple Template");
```

### Key Performance Insights

1. **Simple strings**: Template literals and concatenation perform similarly
2. **With interpolation**: Slight overhead for template literals (negligible in most cases)
3. **Readability wins**: The performance difference is minimal; prioritize code readability
4. **Many concatenations**: Consider `Array.join()` for building strings in loops

### Best Practices

```javascript
// GOOD: Use template literals for readability
const message = `Welcome, ${user.name}! You have ${user.notifications} new messages.`;

// GOOD: Use for dynamic HTML
const html = `<div class="${className}">${content}</div>`;

// AVOID: Don't overuse in performance-critical loops
// Bad
const items = [];
for (let i = 0; i < 10000; i++) {
  items.push(`Item ${i}`);
}

// Better
const items = [];
for (let i = 0; i < 10000; i++) {
  items.push("Item " + i); // Slightly faster
}

// Best
const items = Array.from({ length: 10000 }, (_, i) => `Item ${i}`);
```

## Common Pitfalls

### 1. Using Quotes Instead of Backticks

```javascript
// âŒ WRONG: Using regular quotes
const name = "Arjun";
const message = "Hello, ${name}!";
console.log(message); // Output: Hello, ${name}!

// âœ… CORRECT: Using backticks
const correctMessage = `Hello, ${name}!`;
console.log(correctMessage); // Output: Hello, Arjun!
```

### 2. Forgetting the $ or {}

```javascript
// âŒ WRONG: Missing $
const msg1 = `Hello, {name}!`;

// âŒ WRONG: Missing {}
const msg2 = `Hello, $name!`;

// âœ… CORRECT
const msg3 = `Hello, ${name}!`;
```

### 3. Unintended Whitespace

```javascript
// âŒ Creates unwanted leading/trailing whitespace
const html = `
<div>
  <p>Content</p>
</div>
`;

// âœ… Use trim() or adjust formatting
const cleanHtml = `
<div>
  <p>Content</p>
</div>
`.trim();
```

### 4. Expression Errors

```javascript
const user = null;

// âŒ WRONG: Will throw error
// const msg = `Welcome, ${user.name}!`;

// âœ… CORRECT: Use optional chaining or check
const msg = `Welcome, ${user?.name || "Guest"}!`;
```

### 5. Nested Template Literals

```javascript
// âŒ Hard to read
const msg = `User ${user.active ? `${user.name} (active)` : `${user.name} (inactive)`}`;

// âœ… Better: Extract to variable or function
const status = user.active ? "active" : "inactive";
const msg = `User ${user.name} (${status})`;
```

  const sql = strings.reduce((result, str, i) => {
    return result + str + (i < values.length ? '?' : '');
  }, '');

  return {
    sql: sql.trim(),
    params: values
  };
}

function generateClass(className, properties, methods) {
  const propertyList = properties.map(prop => `    this.${prop} = ${prop};`).join('\n');
  const params = properties.join(', ');
  const methodList = methods.map(method => `
  ${method}() {
    // Implementation for ${method}
    console.log(\`\${this.name} is ${method}ing\`);
  }`).join('\n');

  return `
class ${className} {
  constructor(${params}) {
${propertyList}
  }
${methodList}
}

// Usage example:
const student = new ${className}(${properties.map(p => `"${p}Value"`).join(', ')});
${methods.map(m => `student.${m}();`).join('\n')}
  `.trim();
}

## Summary

Template literals are a powerful ES6 feature that make string handling in JavaScript cleaner and more intuitive. Key takeaways:

1. **Use backticks** (`` ` ``) for template literals
2. **Interpolate with `${}`** for variables and expressions
3. **Multiline strings** work naturally without escape characters
4. **Tagged templates** enable advanced string processing
5. **String.raw** preserves escape sequences for special cases
6. **Performance** is comparable to concatenation; prioritize readability
7. **Common uses** include HTML generation, URL building, logging, and configuration

Master template literals to write more maintainable and professional JavaScript code!
